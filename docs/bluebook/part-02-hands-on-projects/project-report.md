# 项目一·5 分钟搞定工作汇报

📝 你有一段草稿/纪要/零散想法——用 5 分钟变一份能交的汇报。

<div class="tag-row">
  <span class="tag">难度：入门</span>
  <span class="tag">用时：20 分钟</span>
  <span class="tag">WorkBuddy v5.2.5</span>
  <span class="tag rose">项目一</span>
</div>

## 你将拿到什么

- 一份**可交的周报/项目进展**。
- 改写、要点、清单三步法。

## 什么时候用

- 周末写周报没头绪。
- 给老板发项目进展不知道怎么开头。
- 把会议纪要整理成可读的简报。

## 跟着做

### 1. 准备素材（2 分钟）

把以下内容收集到「我的文件/10-工作/本周草稿.md」：

- 本周做过的 3–5 件事（每件事 1–2 句）。
- 遇到的问题（哪怕还没解决）。
- 下周计划（哪怕只是方向）。

### 2. 改写（1 分钟）

打开**新建任务**，贴下面提示词。

```text
角色：你是我（产品经理/运营/工程师/……）的写作助理。

任务：把下面这段零散内容改写成一份周报。

约束：
- 长度：200–300 字
- 语气：正式但不官腔
- 不要：列细节动作，要写"做了什么 + 为什么 + 结果"
- 不要：原话照抄

输出格式：
## 本周完成（3 条）
## 进行中（2 条）
## 下周计划（2 条）
## 需要支持（1 条，可选）

输入：
[粘贴你的零散内容]
```

### 3. 提炼要点（1 分钟）

```text
把上面这份周报压成 3 个 bullet 给我老板发消息用：
- 每条 1 行，不超过 20 字
- 突出"价值"而不是"动作"
```

### 4. 整理成清单（1 分钟）

```text
把上面 3 条 bullet 整理成一份"行动清单"：
- 责任人：___
- 截止时间：___
- 验收标准：___

留空的部分让我自己填。
```

## 验收清单

- [ ] 周报长度 200–300 字，不啰嗦。
- [ ] 包含"完成/进行中/下周/需要支持"四块。
- [ ] bullet 突出价值，不堆动作。
- [ ] 行动清单有可填的字段。
- [ ] 自己再读一遍，确认没有夸大或漏报。

## 常见失败和修复

- ❌ 输出官腔 → ✅ 提示词加"不要原话照抄、要写人话"。
- ❌ 只列做了什么，不说价值 → ✅ 加"突出价值而不是动作"。
- ❌ 一周没产出就空着 → ✅ 把"等待/协调/学习"也算进展，但要明说。

## 练一练

- 用你的真实本周内容跑一次上面的三步。
- 把"周报 + bullet + 清单"存到「作品集 → 01-起步」。

## 作品集节点

存"本周周报 + bullet + 行动清单"到「作品集 → 01-起步 → 周报」。
