# 项目六·把重复流程变成自动化

🤖 把一段已经稳定的流程变成"机器替我做"——4 步走，但要谨慎。

<div class="tag-row">
  <span class="tag">难度：综合</span>
  <span class="tag">用时：45 分钟</span>
  <span class="tag">WorkBuddy v5.2.5</span>
  <span class="tag rose">项目六</span>
</div>

## 你将拿到什么

- 1 段**实际运行**的自动化。
- 1 份**监控清单**：什么时候看、失败了怎么办。

## 什么时候用

- 每周重复做的事 ≥ 3 次。
- 流程已经稳定（不是边做边改）。
- 失败后果可控（不会丢数据/发错地方）。

## 什么时候**不要**做

- 流程还没稳定。
- 涉及删除/对外发送但没有手工确认。
- 数据敏感（敏感操作要人工复核）。

## 跟着做

### 阶段 1：找一段稳定流程（5 分钟）

```text
我每周做 [事]：

1. 触发：[什么时候做]
2. 步骤：
   - 第一步 [____]
   - 第二步 [____]
   - 第三步 [____]
3. 输出：[最终交付物]
4. 失败时：[我通常怎么手动救]

请评估：
- 这段流程是否"稳定到可以自动化"？
- 哪些步骤有判断/例外？
- 风险点是什么？
```

只有当 WorkBuddy 评估为"可自动化"才继续。

### 阶段 2：装技能（10 分钟）

打开**技能**。

```text
我要自动化 [流程描述]。

请推荐 3 个能用的技能：
- 哪个最匹配
- 各自需要什么权限
- 我先装哪个试一下
```

装好后用一次小测试。

### 阶段 3：接连接器（10 分钟）

打开**连接器**。

```text
我的自动化要读写 [系统/工具名]。

请告诉我：
- 该接哪个连接器
- 最小权限是什么
- 同步前我需要做的准备
```

只勾必要权限。

### 阶段 4：建自动化（15 分钟）

打开**自动化**。

```text
新建自动化：

- 名称：[清晰的名字 + 触发时间]
- 触发：[每周一 9:00 / 收到特定邮件 / ……]
- 动作：
  1. 用 [技能名] 处理 [来源]
  2. 通过 [连接器名] 写入 [目标]
  3. 发邮件给 [谁]
- 失败时：发邮件给 [我]
- 手工确认：[开 / 关]

请先列"完整动作链 + 每步的输入输出"，我确认后再上线。
先开测试模式跑 1 周。
```

### 阶段 5：监控（5 分钟）

```text
为这段自动化建一份监控清单：
- 每天/周看什么
- 哪些信号说明出问题
- 怎么停
- 出现问题时找谁

存到 [路径]。
```

## 验收清单

- [ ] 流程评估为"可自动化"。
- [ ] 技能经过测试。
- [ ] 连接器只勾必要权限。
- [ ] 自动化有失败通知 + 手工停开关。
- [ ] 先开测试模式跑 1 周。
- [ ] 监控清单到位。

## 常见失败和修复

- ❌ 自动化放大错误 → ✅ 流程没稳定前不上自动化。
- ❌ 失败没人知道 → ✅ 必加失败通知。
- ❌ 权限过宽 → ✅ 接连接器时只勾必要。
- ❌ 不知道哪个自动化在跑 → ✅ 命名清晰 + tag。

## 练一练

- 选一段你真实重复的流程跑完整 5 阶段。
- 把"自动化配置 + 监控清单"存到「作品集 → 02-项目 → 自动化」。
