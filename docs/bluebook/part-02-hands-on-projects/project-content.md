# 项目三·写一套内容

✍️ 一次产出公众号或视频脚本——5 步走，每步独立可检查。

## 你将拿到什么

- 1 个选题 + 1 个大纲 + 1 份初稿 + 1 张配图建议 + 1 份排版。
- 一套可复用的"内容流水线"提示词。

## 什么时候用

- 周更公众号/小红书/视频号。
- 给公司写一篇技术博客。
- 做一门迷你课的脚本。

## 跟着做

### 1. 选题（5 分钟）

打开**助理**，开一个长对话。

```text
我要做 [平台/类型：公众号长文 / 视频号 60s 脚本 / 小红书图文]。

主题方向：[你最近想聊的话题]
目标读者：[谁会看]
希望他们看完做什么：[点赞 / 关注 / 试用 / 转发]

请给我 5 个选题：
- 每个选题写 1 句"切入角度"
- 标出哪个最有可能被读完
- 标出哪个争议最大
```

确认一个选题后再继续。

### 2. 大纲（10 分钟）

```text
选题：____________________

请写大纲：
- 开头：30 秒/3 行内抓人
- 正文：3–4 个小节，每节 1 句核心
- 结尾：1 句金句 + 1 个行动号召

要求：
- 全文预计 [字数 / 时长]
- 不要写成完整段落
- 每节附 1 个"读者会问的问题"
```

### 3. 初稿（15 分钟）

```text
按上面大纲写初稿。

约束：
- 段落短（手机屏 3 行内）
- 关键句加粗
- 避免"小编认为、大家好"等套话
- 不要 Emoji 堆砌

写完先自检：
- 是不是真的有 1 个观点
- 例子是不是具体
- 标题是不是能让人想点开
```

### 4. 配图建议（5 分钟）

切到 **Design 模式**。

```text
[切到 Design 模式]

这是我的初稿片段（粘贴）：
____________________

请给我 2 张配图方案：
- 1 张主图（封面/首图）
- 1 张内文配图

每张图说明：构图、主体、配色、文案位置。
不要直接出图，先出文字方案。
```

### 5. 排版（10 分钟）

切回 **Work 模式**。

```text
这是初稿：[粘贴]

请按 [平台] 排版规范调整：
- 公众号：每段 3–4 行，关键词加粗，结尾引导关注
- 小红书：标题 ≤ 20 字，正文分段多，标签 5–8 个
- 视频号脚本：标"画面/旁白/字幕"三栏

最后给我一份"发布前 checklist"：
- 标题 OK？
- 开头能抓人？
- 例子具体？
- 结尾有行动？
- 标签齐全？
```

## 验收清单

- [ ] 选题：1 句切入角度，不模糊。
- [ ] 大纲：3–4 节，每节 1 句核心。
- [ ] 初稿：段落短、有具体例子、有 1 个核心观点。
- [ ] 配图：构图/主体/配色说明清楚。
- [ ] 排版：符合平台规范，有 checklist。

## 常见失败和修复

- ❌ 大纲写完就堆成全文 → ✅ 严格按"提纲 → 初稿"两步走。
- ❌ 配图和正文风格不搭 → ✅ 提示词里同时给初稿片段和风格描述。
- ❌ 排版像 Word 文档 → ✅ 提示词加"每段 3–4 行、关键句加粗"。

## 练一练

- 选一个真实主题跑完整 5 步。
- 把"选题 + 大纲 + 初稿 + 配图 + 排版"打包存到「作品集 → 03-内容」。

## 作品集节点

存"内容流水线提示词"到「作品集 → 03-内容 → 提示词库」。
