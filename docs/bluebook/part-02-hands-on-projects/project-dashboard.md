# 项目五·搭一个业务数据看板

📊 不再反复问同事要数字——搭一个能自己更新的业务数据看板。

<div class="tag-row">
  <span class="tag">难度：进阶</span>
  <span class="tag">用时：60 分钟</span>
  <span class="tag">WorkBuddy v5.2.5</span>
  <span class="tag rose">项目五</span>
</div>

## 你将拿到什么

- 1 份**数据看板**（可以是个 Excel/PDF/网页）。
- 1 段**每周自动更新**的脚本或自动化。

## 什么时候用

- 每周要给老板/团队同步业务数字。
- 多个表格需要汇总成一张图。
- 想告别"我去找一下数据"。

## 跟着做

### 阶段 1：确定要看什么（10 分钟）

打开**助理**。

```text
我是一个 [岗位]，每周要给老板看业务数字。

请帮我列出 5–8 个最值得看的指标：
- 指标名
- 怎么算
- 数据从哪来
- 看的频率

不要超过 8 个，先求"少而准"。
```

### 阶段 2：收集数据（10 分钟）

把每个指标的数据源整理到 `我的文件/10-工作/数据看板/原始数据/`：

- Excel/CSV 优先（机器可读）。
- 没有的话，截图也行，但要标"人工"。

### 阶段 3：清洗（10 分钟）

```text
请读 [文件路径] 的 [表名]：

1. 检查字段是否一致：[字段名格式]
2. 找出空值/异常值，列出来
3. 算 [指标]，公式是 [____]
4. 把清洗后的结果写到 [新文件路径]

不要修改原文件，输出版本用日期命名：cleaned_2026-07-14.csv
```

### 阶段 4：图表（15 分钟）

```text
基于清洗后的 [文件]：

请做 3 张图：
1. [指标 1] 趋势（折线）
2. [指标 2 vs 指标 3]（柱状）
3. [指标 4] 占比（饼图/环形）

每张图说明：
- 一句话业务结论
- 异常点
- 需要解释的字段
```

### 阶段 5：报告 + 自动化（15 分钟）

```text
把上面 3 张图整理成一份周报 PDF/网页：
- 顶部 3 个关键数字
- 中间 3 张图 + 一句话结论
- 底部"本周需要关注"（最多 3 条）

再加 1 段自动化提示词：
- 每周一 9:00 触发
- 重新跑清洗 → 重新出图 → 写报告
- 发邮件给 [谁]
```

## 验收清单

- [ ] 5–8 个指标，少而准。
- [ ] 数据有源，路径清楚。
- [ ] 清洗不破坏原数据，输出版本化。
- [ ] 图表带业务结论，不只是图。
- [ ] 自动化有失败通知。

## 常见失败和修复

- ❌ 指标太多 → ✅ 砍到 5–8 个，先求"少而准"。
- ❌ 改了原数据 → ✅ 输出新文件，原文件只读。
- ❌ 图表没结论 → ✅ 提示词里加"一句话业务结论"。
- ❌ 自动化失败没人知道 → ✅ 失败时发邮件/消息。

## 练一练

- 选一个真实业务场景跑完整 5 阶段。
- 把"数据 + 看板 + 报告"打包存到「作品集 → 02-项目 → 数据看板」。

## 作品集节点

存"看板截图 + 报告样例 + 自动化提示词"到「作品集 → 02-项目 → 数据看板」。
