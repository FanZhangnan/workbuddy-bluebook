# 项目四·做一个自己的品牌网站

🌐 一个周末，做出你的个人品牌站——从定位到上线，分 5 步走完。

## 你将拿到什么

- **PB-01 定位简报**：你是谁、为谁、解决什么问题。
- **PB-02 受众报告**：2 类目标用户 + 1 份差异化地图。
- **PB-03 内容架构**：5–7 个页面，每个页面写目标/内容/CTA。
- **PB-04 UI 规格**：配色、字体、组件、响应式规则。
- **PB-05 已上线网站**：可访问的 URL。

> 这 5 个 PB 产物在第 19 章（收官）会再次复用，做你的内容运营系统。

## 什么时候用

- 想有自己的品牌站、博客、作品集。
- 求职/接私活需要一个可发链接的"门面"。
- 经营副业/课程/咨询需要展示页面。

## 跟着做

### 阶段 1：定位 → PB-01（10 分钟）

打开**新建任务**或**助理**。

```text
我想做一个个人品牌网站，名字暂定 [_____]。

请帮我写一份「PB-01 定位简报」：
1. 一句话定位：[我帮 ____ 解决 ____ 的问题]
2. 三个独特优势（结合我的背景）：[背景]
3. 不做什么：[我刻意不做的事]
4. 成功指标（6 个月内）：[3 个可量化指标]

不要给我套话模板，要结合我给的背景。
```

完成后存到 `我的文件/00-作品集/02-项目/品牌站/PB-01-定位.md`。

### 阶段 2：受众 → PB-02（15 分钟）

```text
基于 PB-01：

请给我 2 类目标用户画像：
- 名字 + 职业
- 痛点
- 看到我网站会想做的 1 件事
- 容易不喜欢的 1 类内容

再写 1 份「差异化地图」：
- 同类网站 3 个
- 他们的常见做法
- 我可以刻意做不一样的 2 件事
```

存到 `PB-02-受众.md`。

### 阶段 3：内容架构 → PB-03（15 分钟）

切到 **Plan 模式**。

```text
基于 PB-01 和 PB-02：

请设计 5–7 个页面，每个页面写：
- 页面名
- 目标（访客看完做什么）
- 关键内容
- 行动号召（CTA）

控制总页数：5–7 个，宁少勿多。
```

存到 `PB-03-架构.md`。

### 阶段 4：UI 规格 → PB-04（20 分钟）

切到 **Design 模式**。

```text
风格方向：[温柔/极简/工业/手作 选 1 个]
参考：[粘贴 1–2 个你喜欢的网站截图/描述]
配色：3 个主色 + 2 个辅色
字体：1 个标题字体 + 1 个正文字体
组件：导航/按钮/卡片/表单 至少 4 个
响应式：手机/平板/桌面 三个断点

请先给我 2 套不同风格的方案，不要直接定稿。
```

确认后存到 `PB-04-UI.md`。

### 阶段 5：开发与上线 → PB-05（30 分钟）

切到 **Code 模式**。

```text
按 PB-01..PB-04 实现网站。

要求：
- 静态站（HTML/CSS/JS 或 VitePress/VuePress 等）
- 包含 PB-03 列出的所有页面
- 移动端友好
- 有 sitemap 和 robots
- 不引入未声明的第三方追踪
- 部署到：[Vercel / Netlify / Cloudflare Pages / GitHub Pages]

实现步骤：
1. 先列项目结构
2. 写完每个页面后让我预览
3. 测试手机/平板/桌面
4. 部署后给我 URL
```

## 验收清单

- [ ] PB-01 定位明确，不模糊。
- [ ] PB-02 有 2 类用户画像和差异化地图。
- [ ] PB-03 5–7 页，每页有目标/内容/CTA。
- [ ] PB-04 配色/字体/组件/响应式清楚。
- [ ] PB-05 网站可访问，移动端可用。

## 常见失败和修复

- ❌ 一上来就写代码 → ✅ 先完成 PB-01..PB-04 再开发。
- ❌ 风格飘忽 → ✅ PB-04 选 1 个方向坚持到底。
- ❌ 部署后没人访问 → ✅ PB-01 的成功指标里就包含"6 个月 1000 UV"，上线后立刻做内容运营（第 19 章）。

## 练一练

- 跑完 5 个阶段，得到一个真实可访问的网站。
- 把 5 个 PB 产物 + URL 存到「作品集 → 02-项目 → 品牌站」。

## 作品集节点

存"PB-01..PB-05 + URL"到「作品集 → 02-项目 → 品牌站」。
