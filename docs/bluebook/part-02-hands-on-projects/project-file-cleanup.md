# 项目二·批量整理文件

🗂️ 下载目录里堆了 200 个文件——按主题分、按时间排、改个有意义的名字，再做一份索引。

## 你将拿到什么

- 一个**分好类的文件夹**。
- 一份**索引文件**：每个文件一句话说明。

## 什么时候用

- 电脑下载目录乱成一团。
- 项目结项后需要归档资料。
- 接手同事的工作，要快速看懂文件结构。

## 跟着做

### 1. 拍照现状（2 分钟）

把"待整理的文件夹"截图，记下：

- 总文件数。
- 文件类型分布（截图 Windows 资源管理器按类型分组）。
- 你能想到的"分类方式"（按项目/按时间/按类型）。

### 2. 让 WorkBuddy 提议分类（3 分钟）

打开**助理**，贴：

```text
我要整理 [路径] 下的 [N] 个文件。

文件类型主要有：图片/文档/表格/压缩包……

我倾向于按 [你的分类方式] 分。

请先告诉我：
1. 你建议的 3–5 个分类是什么？
2. 每个分类下大概放哪类文件？
3. 哪些文件可能归类有歧义，需要我手动决定？

不要动手，先讨论。
```

确认分类方案后再开始。

### 3. 改名（10 分钟）

```text
请把 [路径] 下的文件按以下规则改名：
- 格式：[分类]_[简短描述]_[原扩展名]
- 例：发票_2026Q1_XX公司.pdf
- 不要修改扩展名
- 跳过：[我指定要保留原名的文件]

改名前先列"会改哪些文件"，我确认后再改。
```

### 4. 移动到子文件夹（10 分钟）

```text
按上面确认的分类，把 [路径] 下的文件移动到对应子文件夹。
移动前先列"每个文件会移动到哪里"。
出现以下情况时停下来问我：
- 文件名重复
- 移动会覆盖已有文件
- 分类不明确
```

### 5. 做索引（5 分钟）

```text
为 [路径] 做一个索引文件 `00-索引.md`：
- 列出每个子文件夹
- 每个子文件夹下，列文件 + 一句话说明（读文件名推断）
- 我没确认的内容标注"未确认"

不要靠"印象"猜文件内容。
```

## 验收清单

- [ ] 文件按确认的分类分到子文件夹。
- [ ] 命名格式一致，没有空格、没有奇怪字符。
- [ ] `00-索引.md` 在最外层。
- [ ] 没覆盖原文件（移动前有确认步骤）。
- [ ] 重要文件备份过（整理前先 cp 一份到 `_backup`）。

## 常见失败和修复

- ❌ 分类太细 → ✅ 控制在 3–5 个，再细就两层。
- ❌ 改名后找不到原文件 → ✅ 整理前先备份到 `_backup_2026-07-14`。
- ❌ 索引不准确 → ✅ 让 WorkBuddy "不要靠印象猜"。

## 练一练
