# 模式

🎛️ WorkBuddy 默认是 **Work 模式**。但它还有其他模式：Code、Design、Plan。模式选错，事倍功半。

<div class="tag-row">
  <span class="tag">难度：入门</span>
  <span class="tag">用时：10 分钟</span>
  <span class="tag">WorkBuddy v5.2.5</span>
</div>

## 你将拿到什么

- 一张「模式选择清单」：看到任务就知道用哪个模式。
- 4 个模式的典型场景。

## 四个模式快速对比

| 模式 | 适合 | 不适合 |
|---|---|---|
| **Work** | 写文档、改稿、做总结、查资料、分析数据 | 写代码、做 UI 图 |
| **Code** | 写网站、做脚本、跑命令、生成代码 | 不写代码的纯文字工作 |
| **Design** | 画 UI 草图、生成 logo、配色方案 | 写代码或写长文 |
| **Plan** | 复杂任务规划、方案设计、跨阶段拆解 | 一次性小问答 |

## 什么时候用哪个

- **Work 模式（默认）**：99% 的日常办公任务都先选它。
- **Plan 模式**：要做一件大事、写一个方案、拆一个项目。Plan 模式会主动帮你分阶段。
- **Design 模式**：想要 UI 草图、配色、icon、logo。生成的是设计稿，不能直接当代码用。
- **Code 模式**：要写一个网站、跑一段脚本。Code 模式会改你的项目文件，慎用。

## 三步搞定

1. 看主对话区顶部的**模式徽标**，确认现在是 Work。
2. 想换模式时，点模式名或在顶栏切换。
3. 切换后**第一句话要把任务说完整**——别让它猜。

## 提示词：Plan 模式开项目

```text
[切到 Plan 模式后]

我想做：____________________

请先别动手做，帮我：
1. 拆成 3–5 个阶段；
2. 每个阶段列目标和产物；
3. 标出关键风险。

我确认后再开始。
```

## 提示词：Design 模式画草图

```text
[切到 Design 模式后]

我要做一个：[网站 / 海报 / 海报图] 的草图。

风格参考：[粘贴 1–2 个参考图/描述]
受众：[谁会看]
必含元素：[logo 位置、文案、行动按钮]

请先给我 2 个不同方向的草图，不要直接定稿。
```

## 常见失败和修复

- ❌ 在 Work 模式里让它写代码 → ✅ 切到 Code 模式；Work 模式不擅长改项目文件。
- ❌ Design 模式生成的设计稿直接当代码用 → ✅ 草图确认后切到 Code 模式开发。
- ❌ Plan 模式自动开始动手 → ✅ 提示词里加"先别动手做"。

## 练一练

- 找一个写文档的任务在 Work 模式做，体会默认模式。
- 切到 Plan 模式拆一个真实项目。
- 切到 Design 模式给品牌站画一张首屏草图。

## 作品集节点

存 4 个模式各一次的截图到「作品集 → 模式对比」。
