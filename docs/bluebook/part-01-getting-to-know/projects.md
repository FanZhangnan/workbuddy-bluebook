# 项目

📂 持续好几天、有多个产物、需要反复打开的事——用一个**项目**装起来，比开一堆会话更省心。

<div class="tag-row">
  <span class="tag">难度：入门</span>
  <span class="tag">用时：12 分钟</span>
  <span class="tag">WorkBuddy v5.2.5</span>
</div>

## 你将拿到什么

- 一个「项目模板」：目标、阶段、产物、资料、风险。
- 知道什么时候新建任务、什么时候开助理、什么时候开项目。

## 什么时候用

- 做一个产品上线活动（要写方案、做海报、发邮件、跟进反馈）。
- 给一个客户做调研报告（要查资料、写报告、做汇报 PPT）。
- 写一本书/做一门课（章节多、产物多、要持续几周）。

## 项目 vs 助理 vs 任务

- **任务**：一次性问答，用完即走。
- **助理**：一段持续对话，记忆靠对话本身。
- **项目**：**多阶段 + 多产物 + 共享资料**。WorkBuddy 会把每个阶段的产物串起来。

## 三步搞定

1. 侧栏点「项目 → 新建项目」，给项目起个名字。
2. 第一轮就让 WorkBuddy 帮你拆阶段、列产物、找风险。
3. 之后每个阶段产出一份文件，挂在项目里，方便以后查。

## 提示词：开一个项目

```text
我要开一个项目：____________________

请帮我做三件事：
1. 拆成 3–5 个阶段，每个阶段说明目标和产物；
2. 列出需要的资料（我已经有的 / 需要找的）；
3. 标出可能卡住的地方和备选方案。

阶段命名格式：「阶段 N：名字（产物）」
```

## 项目模板

每个项目都建议保留这 5 块：

```text
# 项目名

## 目标
- 一句话讲清楚要做成什么

## 阶段
- 阶段 1：……（产物：……）
- 阶段 2：……（产物：……）
- 阶段 3：……（产物：……）

## 资料
- 我有的：……
- 我要找的：……

## 风险与备选
- 风险 A：…… → 备选：……
- 风险 B：…… → 备选：……

## 版本与状态
- 当前阶段：……
- 更新时间：……
```

## 常见失败和修复

- ❌ 阶段拆得太细，每步都问一遍 → ✅ 阶段控制在 3–5 个，再细就在阶段内开助理。
- ❌ 资料散落在多个对话里 → ✅ 用"我的文件"或"知识库"统一存，项目里只放引用。
- ❌ 项目变成长对话的堆叠 → ✅ 阶段结束就小结一次，关掉旧助理，开新阶段。

## 练一练

- 用上面提示词开一个"做一个自己的品牌网站"项目，让 WorkBuddy 帮你拆阶段。
- 把现有的一段工作拆成 3 个阶段。
- 试着在不同阶段开新助理，看看上下文是否延续。

## 作品集节点

保存项目模板到「作品集 → 项目模板」。
