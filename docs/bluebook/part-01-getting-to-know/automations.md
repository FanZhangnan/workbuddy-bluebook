# 自动化

⏰ 「自动化」让一段流程在某个条件下自动跑。可以是定时、可以是事件触发。

<div class="tag-row">
  <span class="tag">难度：进阶</span>
  <span class="tag">用时：10 分钟</span>
  <span class="tag">WorkBuddy v5.2.5</span>
</div>

## 你将拿到什么

- 一份「什么时候上自动化」判断清单。
- 3 个常见自动化模式。

## 什么时候用

- 每周要发一份周报。
- 每天要汇总某个群/邮箱里的信息。
- 某个事件触发后要做一连串事（新建任务 → 发邮件 → 更新表格）。

## 什么时候**不要**用

- 流程还没稳定——自动化只会放大错误。
- 涉及对外发送/删除——必须有手工确认环节。
- 触发条件不明确——会产生误报或漏报。

## 三步搞定

1. 侧栏点「自动化 → 新建自动化」。
2. 设**触发条件**（定时/事件）、**动作**、**最后做什么**。
3. **第一次用"测试运行"**，不要直接上线。

## 提示词：设计自动化

```text
我要做一段自动化：

触发：[每周一 9:00 / 收到特定邮件 / 表格新增一行]
动作：
  1. 读 [来源]
  2. 处理 [方式]
  3. 写 [目标]
最后：发邮件给 [谁] / 不通知

请先列出：
- 这个自动化的"危险点"是什么？
- 失败时怎么提醒我？
- 我应该先手工跑几次再上线？
```

## 3 个常见自动化模式

### 模式 1：定时汇总

```text
触发：每周一 9:00
动作：读上周邮件 → 汇总成 5 条要点
最后：写到「我的文件」的"周报"文件
```

### 模式 2：触发式通知

```text
触发：特定邮件到达
动作：抽取关键字段（发件人/主题/金额）
最后：发到工作群
```

### 模式 3：表格驱动

```text
触发：表格新增一行
动作：检查字段是否完整
最后：完整则处理；不完整则标红 + 通知负责人
```

## 常见失败和修复

- ❌ 触发条件太宽，频繁误报 → ✅ 加过滤条件；先观察一周。
- ❌ 自动化跑失败没人知道 → ✅ 失败时发邮件/消息通知自己。
- ❌ 想停却找不到开关 → ✅ 建自动化时给它起清晰名字，并加 tag。

## 练一练

- 找一段你每周重复做的事，写成"触发 + 动作 + 通知"。
- 用"测试运行"跑一次，记录输出。
- 决定哪些条件可以上线，哪些还需要手工。

## 作品集节点

存"我的自动化清单"到「作品集 → 自动化」。
