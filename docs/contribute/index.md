# 参与共创

正式课程、社区案例和模板分开治理。先阅读仓库根目录 `CONTRIBUTING.md` 与行为准则，再选择投稿类型。

## 可贡献内容

- **事实纠错：** 提供原文位置、官方依据、影响与建议改法。
- **社区案例：** 使用真实但已授权、脱敏的情境，包含完整提示词、结果证据、失败和复核。
- **模板改进：** 说明变量、使用示例、关联章节、版本与许可证。
- **正式章节提案：** 先提出学习成果和验证方式；正式内容由编辑团队重写维护。

## 审核标准

不得虚构 WorkBuddy UI、连接器或版本；未实测能力写“以当前官方版本为准”。不得提交密钥、个人信息、未授权客户资料和私密截图。所有写入、发送、发布、删除动作必须标人工确认点。

## 本地验证

```bash
npm ci
npm run check
```

使用仓库要求的 Node 22 managed runtime。PR 必须通过课程编号/路由、MVP 文件、frontmatter 与生产构建检查。

## 内容许可证

代码采用 MIT；原创正文与下载模板采用 CC BY 4.0。第三方素材必须逐项标来源与许可证，不确定时不要提交。
