# 参与贡献

感谢你改进 WorkBuddy 蓝皮书。代码采用 MIT 许可证；原创正文与下载模板采用 CC BY 4.0。

## 可贡献内容

- **事实纠错：** 标明页面、受影响表述、官方证据和建议改法。
- **社区案例：** 使用已授权、已脱敏的输入，并包含提示词、证据、验收、失败、人工复核、权限和版本边界。
- **模板：** 声明变量、关联章节、版本和许可证。
- **正式章节提案：** 先提交学习目标和验证方式；最终文案由编辑团队维护。

不得提交密钥、个人信息、私密截图或未经授权的客户材料。不得虚构 WorkBuddy 界面、连接器、版本或价格。未实测的产品行为统一写“以当前官方版本为准”。对外发送、发布、删除和覆盖必须保留人工确认。

## 本地检查

使用 Node 22.22.2，然后运行：

```bash
npm ci
npm run check
```

在本地安全沙箱中，构建工具清理临时输出可能触发宿主的批量文件保护；应按宿主提示确认构建缓存操作，不要通过脚本绕过保护。

Pull Request 必须通过课程编号/路由唯一性、已发布章节 frontmatter、下载清单和生产构建检查。
